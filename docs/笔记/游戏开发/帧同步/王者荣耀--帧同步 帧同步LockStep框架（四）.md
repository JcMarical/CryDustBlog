# 一 帧同步简介
## 帧同步特点
- 帧同步：**只转发操作**
- 游戏逻辑都在客户端实现，服务器主要负责**广播和验证**操作
- 逻辑易实现，数据量少，可重播。
- “慢”优势（严格帧同步）&乐观帧同步：简单一点
- 有利于回放、观战（状态同步存快照也可以做到）
- 容易作弊,安全性差
- 同步问题具有挑战性
- 定点数
- 大端小端问题
- 伪随机数
## 状态同步特点
- 战斗逻辑在服务端
- 每次改变：同步一次属性（帧同步不需要）
- 安全性好
## 帧同步原理
- 逻辑帧：时间片划分，目标是同步所有客户端
- C/S架构：服务端转发操作，客户端执行逻辑

# 二 定点数与浮点数
Unity实现值变量其实也是依靠的结构体来的，要继承几个接口来使用.
- IEquatable:需要重写Equal判断相等的方法。
- IComparable: 需要重写排序（大于小于）的方法。
